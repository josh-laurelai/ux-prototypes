Laurel Design System — Agent Instructions

This file is the complete component reference for building Laurel prototypes. Follow these rules before writing any HTML.

  1. Use only the components and patterns shown in this file. Do not invent new classes, color combinations, or layouts.
  2. Tailwind classes only — from the Laurel palette. Colors are overridden in the config. bg-blue-600 = #39B2FC. No arbitrary hex values.
  3. Typography: Inter font, specific weights only. Use font-normal (body), font-medium (subtitle), font-semibold (title), font-bold (subheadline), font-extrabold (headline).
  4. If a component you need is not shown here, use the closest match. Do not improvise new variants.
  5. Default border radius is rounded-lg (8px). Use rounded-xl for list containers and panels. Use rounded-full only for avatars and filter pills.
  6. Standard content padding: p-4 inside panels, px-4 py-3 for rows and headers.

Quick component guide

  • Chip — status, activity tags, billing codes, filters, counts
  • Filter Chip Row — horizontal pill filters (active = filled dark)
  • Button — actions only; pick variant by importance
  • Toolbar / Action Bar — checkbox count + icon actions + primary button above a list
  • Entry Row — time entries inside an initiative group; increment + status on the right
  • Activity Row — captured browser/app events in the work activity panel
  • Suggested Matters — AI suggestions: light gray box, sparkle header, pill chips
  • Stat Card — key metric: hours, count, total, percentage
  • App Bar Nav Item — icon-only items in the left black nav strip
  • Data Table — tabular data with column headers
  • Empty State — zero-data screen with optional CTA

Chip color guide

  • gray — draft, neutral, default state
  • blue — informational, in-progress, synced
  • green — approved, submitted, positive
  • dark-green — billed, finalized, strong positive
  • yellow — pending, needs review, caution
  • orange — warning, action required
  • red — error, rejected, deleted
  • purple — AI-generated, system feature
  • white/outlined — billing codes, neutral labels

Button color guide

  • black — default action (New Entry, Save, Submit)
  • outline — all secondary/other actions
  • green — Release ONLY (billing release action)
  • ghost (no bg) — cancel, dismiss
