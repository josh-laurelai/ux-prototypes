Variant A
Visual De-emphasis + Text Status Badges
The lightest-touch change: released entries fade to ~45% opacity so they stop competing for attention. Status badges use text labels (not just colored dots) so meaning is unambiguous. Entries needing action get a subtle warm background tint.

Tradeoffs

✓ Pros Lowest implementation cost. Doesn't change page architecture or sorting. Released items naturally fade from view while staying accessible on hover. Text badges eliminate color-only interpretation.
✗ Cons Still requires users to scan the full list — no aggregation of "what needs my attention." If many entries are ready-to-release across many matters, users must scroll through everything. Progress bar helps but is small.
Variant B
Status Sub-groups Within Matters
Within each matter, entries are grouped by status: errors first, then ready-to-release, then released (collapsed by default). Each sub-group has a labeled header so the structure is immediately scannable. This preserves matter grouping while adding status hierarchy inside.

Tradeoffs

✓ Pros Clear hierarchy within each matter — errors surface first, then actionable items, then completed. Released entries can be collapsed. Inline "Release" buttons reduce friction. Fully-released matters can be collapsed to one line.
✗ Cons Adds visual complexity within each matter card. May conflict with user's custom entry ordering (duration, etc.) since status grouping overrides it. More vertical space per matter when multiple status groups exist.
Variant C
Action Summary Bar + Progressive Disclosure
A persistent summary bar surfaces what needs attention without requiring any scanning. Released entries are collapsed into a "show N released" toggle at the bottom of each matter. Matter cards get a left-border accent indicating their aggregate status.

Tradeoffs

✓ Pros Answers "what do I need to do?" instantly — no scanning required. Released entries are hidden by default, radically reducing visual noise. Left-border accents make matter-level status glanceable. "Release all" in the summary bar enables one-click batch action.

Variant D
Focus Tabs — Separate Views by Status
Replace the filter dropdown with persistent tabs that split the timesheet into focused views. "Needs Attention" becomes the default landing tab, showing only entries that require action. This is the most aggressive restructuring — it changes the mental model from "list of everything" to "show me what matters."
