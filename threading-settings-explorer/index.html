What this is: Current production UI. Gear icon opens a separate modal for per-type settings (Documents, Emails, Teams). Three-way toggle and per-type config are in different surfaces.
Problems:
• Per-type settings live behind a gear icon in a separate modal — most users won't find them
• You can set Documents=ON in the modal but display=Off here — contradictory state
• Gear icon suggests it configures the toggle, but it actually configures something orthogonal
• Three-way toggle mixes two decisions: "do I want threading?" and "how should threads display?"

Alt 4: Inline Types
Master toggle on/off. When on, per-type toggles and display mode appear inline. Everything in one surface, no hidden modals.
What it solves:
• Single on/off gate — no contradictory states
• Per-type config is always visible when threading is on
• Display mode is a secondary choice, not mixed into the primary toggle
• Teams sub-categories accessible via dropdown without taking extra vertical space
Tradeoffs:
• Popover is tall when threading is on — three type rows + display mode + filters
• All per-type toggles are always visible — is that overwhelming, or is it the right level of control?
• No way to "partially thread" (e.g., thread Docs but not display threads for Emails) — is that a real need?

Alt 5: Chip Toggles
Same master toggle, but per-type config uses compact pill chips instead of full rows. More space-efficient — the whole threading section takes less vertical room.
What it solves:
• Same on/off gate as A, but more compact per-type control
• Chips make the active/inactive state very scannable
• Leaves room for more types without the popover becoming unwieldy
Tradeoffs:
• Chips are less descriptive — no room for subtitle text like "Similar file names"
• The Teams chip + dropdown combo is a bit unusual — is it clear that you tap the chip to toggle Teams entirely vs the dropdown to customize sub-types?
• Dark chips might feel heavy in a settings surface
Try it: Toggle threading on/off. Click chips to include/exclude types. Use the Teams dropdown for sub-categories.

Alt 1: Threads (Full Config)
Master toggle gates everything. When on, "What to thread" and "How to display threads" appear as two labeled sub-sections — all config always visible.
What it solves:
• Makes the two orthogonal decisions explicit: what types participate, and how threads render
• The section labels ("What to thread" / "How to display") teach the mental model
• Master toggle eliminates contradictory states entirely
Tradeoffs:
• Most structured — might feel heavy for a "just turn it on" user
• The sub-section labels add information architecture that's only useful on first visit — after that they're just noise
• Is the divider between "what" and "how" earning its space, or would they read fine as one continuous section?
Try it: Toggle threading on/off. The entire config section appears/hides. Switch Linked/Collapsed to see the description update.

Alt 3: Groups (Disclosure)
Same "groups" language as Alt 2, but per-type config is hidden behind a "Customize" disclosure link instead of always visible.
What it solves:
• Shortest default state — toggle + display mode is only two controls
• "Customize which types are grouped" is accessible but doesn't overwhelm
• Prioritizes the decision most users care about (threaded vs expanded) over the one most don't (which types)
Tradeoffs:
• Per-type config is hidden — users who need it might not find it
• If a user turns off Docs grouping, there's no visible indicator in the collapsed state
• Compared to the full Groups variant: less upfront info, more compact, but less discoverable
Try it: Toggle grouping on/off. Switch Threaded/Expanded. Click "Customize" to reveal per-type toggles.
