Goals Feature — Hide/Reveal

Exploring how to tuck the weekly/daily hour targets behind an icon to declutter the toolbar for the majority of users.

Current state: Week and Day goals are always visible, taking up ~200px of toolbar space. The numbers lack context — what do 13.4 and 12.2 mean? Actual vs target? Which is which? The colored dots add more mystery.

Variant A — Inline Toggle: A target icon replaces the goals numbers. Click it to expand the numbers inline. Click again to collapse. The icon has a subtle status dot so goal users can see at-a-glance if they're over/under without expanding. Try clicking the icon.

Tradeoffs

Pros

Minimal disruption — same info, same location. Status dot gives ambient awareness without expanding. Toolbar stays a single row. Very fast to implement.

Cons

Still shows the same confusing "two numbers" pattern when expanded. No room to improve the goals display (add context, progress bars, edit targets). Toolbar can feel cramped when expanded on narrow screens.

Variant B — Popover: Icon button opens a small dropdown with labeled goal data. More room to show actual vs target clearly. Click the icon to toggle.

Tradeoffs

Pros

Clean toolbar — icon only. Popover gives room for clear labels, delta, and "over/under" context that the current inline display lacks. Foundation for richer features (edit goals, progress bars, period selection). Familiar pattern (Slack, Linear all use icon + popover for secondary info).

Cons

Requires a click to see numbers — slightly worse for the few power users who glance at them constantly. Popover must be positioned carefully to not overlap the Release button.

Variant C — Popover + Progress Bars: The richer version. Visual progress toward targets with color-coded bars. This is where hiding the feature actually improves it — more room means better data viz. Click the icon.

Room for contextual info like "2 days left." Foundation for editing targets, switching periods, or linking to reports. The feature might actually get used if it's this clear.

Cons

More design/dev work upfront. Need to handle edge cases (no goals set, start of week, targets exceeded by 200%). Popover positioning on small screens. Adds scope if the plan was "just hide it."

Icon Comparison

Which icon best communicates "hour targets" to a lawyer who's seen it zero times?

Target / Bullseye

Best semantic fit — "targets" and "goals" are synonyms. Universal metaphor. Used by Notion, Asana for goals. Might read as "focus" or "record" to some users.

Recommended

Bar Chart

Connotes "stats" and "metrics" broadly. Very familiar. But might be confused with a reporting/analytics feature rather than personal hour targets.

Gauge / Speedometer

Conveys "progress" and "how far along am I." Dynamic feeling. Less common in productivity tools. Could be confused with performance/speed metrics.

Other candidates worth considering

Bolt / Zap

Energy, pace

Pulse / Activity

Tracking, health

Sliders / Levels

Adjustable targets

Checklist / Clipboard

Goals as tasks

Recommendation: Target / Bullseye

The target icon has the strongest semantic mapping to "goals" — users in usability tests consistently associate it with targets and objectives. It's the icon Linear, Notion, and Asana use for goal-tracking features. The bar chart is a close second but risks confusion with reporting. Avoid the gauge — it's more "dashboard widget" than "personal target."
