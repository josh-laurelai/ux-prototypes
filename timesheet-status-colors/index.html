Released entries genuinely disappear from your attention. Red stays for errors. No green anywhere — it's not needed.

Color System 1 — Tradeoffs

✓ Strengths Orange/amber is the strongest "act now" signal that isn't red. Grey is the most neutral possible — released entries genuinely vanish from attention. Clean two-tone system (warm = action, cool/neutral = done). Works well for colorblind users since it doesn't rely on red/green distinction.
✗ Risks Orange can read as "warning" rather than "ready" — users might hesitate to release because the color feels cautionary. No positive reinforcement for released state (grey feels like absence rather than accomplishment). Lawyers may associate orange with problems.
Color System 2
Blue Action / Muted Sage
Blue = "this is your primary action" — same color as buttons and links. Users already associate blue with "click me." Muted sage/green-grey = "done" — a soft, earthy neutral that carries a subtle hint of "complete" without the vibrancy of full green. This system borrows from Jira's model where blue = active work.

Color System 2 — Tradeoffs

✓ Strengths Blue is the universal "interactive/clickable" color — users instinctively know to act on blue things. Muted sage keeps a whisper of "done" feeling without the vibrancy of green. Clean separation between action (warm blue), error (orange), and complete (neutral). Most professional-feeling palette for legal software.
✗ Risks Blue for "ready" might compete with other blue interactive elements (links, buttons) in the UI. Sage-grey might be too subtle — users may not read it as "complete" at first. If your product already uses blue heavily, "ready" entries could blend into the chrome rather than standing out.
Color System 3
Green = Go (Act!) / Purple = Done (Processed)
Leans into the traffic-light metaphor correctly: green = go, act on this NOW. Borrows GitHub's "merged = purple" pattern for released entries — purple reads as "processed, filed away, inert." It's not a warning, it's not positive — it's simply over. Combined with opacity reduction, purple entries fade into a quiet background layer.

Color System 3 — Tradeoffs

✓ Strengths Aligns with traffic-light instinct correctly — green genuinely means "go, act now." Purple for "done" is novel and distinctive — users won't confuse it with any action state. GitHub has normalized purple-for-merged with millions of developers. Combined with opacity, purple fades beautifully into background noise.
✗ Risks Green for "ready" is a semantic swap from today — released entries used to be green, so this is a 180° change that could confuse existing users. Purple for "released" is unconventional in legal/billing tools. Users may need a learning period. Some may read purple as "special" rather than "inert."
